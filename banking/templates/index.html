IndyKite Banking

Use Knowledge-Based Access Control (KBAC) and ContX IQ (CIQ) to retrieve or upsert banking data according to context. CIQ provides fine-grained, context-aware data access for users, accounts, transactions and branches.

How ContX IQ Works
1. Policy

Defines what data can be accessed and who can access it. Uses Cypher patterns to match graph structures and filters to bind parameters. Specifies allowed reads, upserts, and deletes.

2. Knowledge Query

Defines what data to return from the matched graph. Specifies which node properties and relationships to include in results.

3. Execute

Runs the query with input parameters. The policy validates access, the query retrieves data, and results are returned. Uses the application's credentials for authorization.

ContX IQ Use Cases
1 Account Statement Access
Policy Query Execute

Get a user's own FinAccount details (account number, type, balance, opening date). Verifies the caller actually owns the account.

Reads: caller info, owned FinAccount details
Graph Pattern: (caller:User)-[:OWNS]->(account:FinAccount)
Input: caller_id
2 Recent Transactions Lookup
Policy Query Execute

For a given account, return recent transactions (date, amount, source). Caller must own the account.

Reads: account number, transaction date / amount / source
Graph Pattern: (caller:User)-[:OWNS]->(account:FinAccount)-[:HAS_TRANSACTION]->(tx:Transaction)
Input: caller_id, account_external_id
3 Branches used by an Organization's members
Policy Query Execute

List the branches where users that are members of a given organization work.

Reads: organization name, user name, branch code / name / address
Graph Pattern: (user:User)-[:MEMBER_OF]->(org:Organization), (user)-[:WORKS_FOR]->(branch:Branch)
Input: org_id
